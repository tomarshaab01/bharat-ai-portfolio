Project — 01
Handwriting Recognition
A CNN-based system trained on the MNIST dataset to classify handwritten digits 0–9 with high accuracy using deep learning.
View on GitHub ← Back to Portfolio
✓ Complete
🎯
Objective
What it does
Trains a Convolutional Neural Network on the MNIST dataset (60,000 training images) to classify handwritten digits 0–9. The model takes a 28×28 grayscale image as input and outputs the predicted digit with a confidence score.
Image Classification CNN MNIST
🏗️
Architecture
Model Design
Two Conv2D layers (32 and 64 filters) with ReLU activation, followed by MaxPooling layers for spatial downsampling, a Flatten layer, a Dense hidden layer of 128 units, Dropout (0.5) for regularisation, and a softmax output layer for 10 classes.
Conv2D MaxPooling Dropout Softmax
⚙️
Training
How it was trained
Trained using the Adam optimiser with sparse categorical cross-entropy loss. Pixel values normalised to [0,1] range. Ran for 10 epochs with a batch size of 128. Training and validation accuracy tracked per epoch using Matplotlib plots.
Adam Optimiser 10 Epochs Batch 128
📊
Results
Performance
Achieved ~99% training accuracy and ~98.8% test accuracy on the MNIST test set (10,000 images). Confusion matrix generated to identify which digit pairs were hardest to separate. Model saved as .h5 for reuse.
~99% Train Acc ~98.8% Test Acc Confusion Matrix

handwriting_model.py
from tensorflow.keras import layers, models
from tensorflow.keras.datasets import mnist

# Load & normalise MNIST
(X_train, y_train), (X_test, y_test) = mnist.load_data()
X_train = X_train.reshape(-1, 28, 28, 1) / 255.0
X_test = X_test.reshape(-1, 28, 28, 1) / 255.0

# Build CNN
model = models.Sequential([
  layers.Conv2D(32, (3,3), activation='relu', input_shape=(28,28,1)),
  layers.MaxPooling2D(),
  layers.Conv2D(64, (3,3), activation='relu'),
  layers.MaxPooling2D(),
  layers.Flatten(),
  layers.Dense(128, activation='relu'),
  layers.Dropout(0.5),
  layers.Dense(10, activation='softmax')
])
model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(X_train, y_train, epochs=10, batch_size=128, validation_split=0.1)
Project — 02
Sign Language Detection
A real-time hand gesture recognition system using MediaPipe Hands landmark detection to classify ASL alphabet signs via a webcam feed.

Objective
What it does
Detects and classifies American Sign Language (ASL) hand gestures in real-time using a webcam. MediaPipe extracts 21 hand landmarks per frame, which are fed into a classifier to predict the corresponding letter or gesture, with the result overlaid on the live video feed.
Real-Time Detection ASL Gestures Webcam
✋
Landmark Tracking
MediaPipe Pipeline
MediaPipe Hands detects and tracks 21 keypoints on the hand (fingertips, knuckles, wrist) per frame. Each landmark provides (x, y, z) coordinates, giving 63 normalised features per hand. These features are extracted in real-time without any GPU requirement.
21 Landmarks 63 Features x/y/z Coords
⚙️
Classification
How gestures are classified
Landmark coordinates are flattened into a feature vector and passed to a Random Forest classifier (or a small Dense neural network). A custom dataset of per-gesture landmark samples is collected using the webcam capture script, labelled, and used for training.
Random Forest Custom Dataset Feature Vector
🖥️
Live Overlay
OpenCV Output
OpenCV captures the webcam stream frame-by-frame. MediaPipe's drawing utilities render the hand skeleton (bones and joints) directly on the frame. The predicted gesture label is displayed as a text overlay with confidence score, running at ~30 FPS on CPU.
OpenCV ~30 FPS CPU Only
Tech Stack
