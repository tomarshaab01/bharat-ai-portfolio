What it does

Core Features

📐
Plane Detection
Real-time horizontal surface detection using ARCore raycasting. Scans room geometry and highlights flat surfaces as valid placement areas.
✋
Multi-Touch Gestures
Pinch to scale, two-finger twist to rotate, single-tap to place. Full 3-axis manipulation of placed objects in real space.
⚡
Performance Optimised
LOD-adjusted prefabs and efficient update loops maintain 60fps on mid-range Android. Draw calls kept under 50 per frame.
🎯
Persistent Anchors
Objects stay locked to real-world surfaces using AR anchors. Camera can move freely — objects don't drift or float.
📦
Object Selector UI
Bottom-drawer UI panel lets users pick from multiple 3D object types before placing. Supports chairs, tables and decorative items.
🗑️
Delete & Undo
Tap any placed object to select it, then delete with a button. Full undo stack using a command pattern for action reversal.

How it works

Technical Flow

01
ARSession & Camera Init
AR Foundation starts an ARSession, activates the device camera via ARCameraManager, and initialises the ARPlaneManager for surface tracking. The session runs its own update loop separate from Unity's main thread.
02
Plane Detection & Visualisation
ARPlaneManager fires events when planes are added, updated or removed. Each plane spawns a mesh visualiser prefab showing detected boundaries with an animated grid overlay.
03
Raycast Hit & Placement Indicator
Every frame, ARRaycastManager fires a ray from screen-centre against TrackableType.PlaneWithinPolygon. On stable hit, a reticle indicator snaps to the surface normal, giving the user clear visual feedback before placement.
04
Object Instantiation with ARAnchor
On tap, the selected prefab is Instantiated at the hit pose. An ARAnchor component is added at that point — ARCore then tracks this anchor against the physical world so the object stays pinned even during camera movement.
05
Gesture Recognition & Transform
GestureManager monitors Input.touchCount each frame. Two-finger pinch delta drives localScale. Rotation delta (angle between touch vectors) drives Y-axis Euler rotation. Single-finger drag moves the selected object along the plane normal.

using UnityEngine;
using UnityEngine.XR.ARFoundation;
using UnityEngine.XR.ARSubsystems;
using System.Collections.Generic;

[RequireComponent(typeof(ARRaycastManager))]
public class PlacementController : MonoBehaviour {

  [SerializeField] private GameObject placementIndicatorPrefab;
  private ARRaycastManager raycastManager;
  private ARAnchorManager anchorManager;
  private GameObject indicator;
  private bool isPlacementValid = false;
  static List<ARRaycastHit> hits = new List<ARRaycastHit>();

  void Awake() {
    raycastManager = GetComponent<ARRaycastManager>();
    anchorManager = GetComponent<ARAnchorManager>();
    indicator = Instantiate(placementIndicatorPrefab);
    indicator.SetActive(false);
  }

  void Update() {
    var screenCenter = new Vector2(Screen.width * 0.5f, Screen.height * 0.5f);
    isPlacementValid = raycastManager.Raycast(
      screenCenter, hits, TrackableType.PlaneWithinPolygon);

    indicator.SetActive(isPlacementValid);
    if (isPlacementValid) {
      indicator.transform.SetPositionAndRotation(
        hits[0].pose.position,
        hits[0].pose.rotation);
    }
    // Tap to place
    if (isPlacementValid && Input.touchCount == 1 &&
        Input.GetTouch(0).phase == TouchPhase.Began) {
      PlaceObject(hits[0].pose);
    }
  }

  void PlaceObject(Pose pose) {
    var prefab = ObjectSelector.Instance.GetSelected();
    var obj = Instantiate(prefab, pose.position, pose.rotation);
    // Attach anchor so object stays in world space
    var anchor = anchorManager.AttachAnchor(
      hits[0].trackable as ARPlane, pose);
    obj.transform.parent = anchor.transform;
    UndoStack.Instance.Push(obj);
  }
}
using UnityEngine;

public class GestureManager : MonoBehaviour {

  private GameObject selectedObject;
  private float prevPinchDist = 0f;
  private float prevAngle = 0f;
  const float SCALE_SPEED = 0.01f;
  const float MIN_SCALE = 0.1f;
  const float MAX_SCALE = 3.0f;

  void Update() {
    if (Input.touchCount == 2) {
      HandleTwoFingerGesture();
    } else if (Input.touchCount == 1) {
      HandleSingleTap();
    } else {
      prevPinchDist = 0f;
    }
  }

  void HandleTwoFingerGesture() {
    if (selectedObject == null) return;
    Touch t0 = Input.GetTouch(0);
    Touch t1 = Input.GetTouch(1);

    // Pinch to scale
    float dist = Vector2.Distance(t0.position, t1.position);
    if (prevPinchDist > 0f) {
      float delta = (dist - prevPinchDist) * SCALE_SPEED;
      float newScale = Mathf.Clamp(
        selectedObject.transform.localScale.x + delta,
        MIN_SCALE, MAX_SCALE);
      selectedObject.transform.localScale =
        Vector3.one * newScale;
    }
    prevPinchDist = dist;

    // Twist to rotate
    Vector2 dir = t1.position - t0.position;
    float angle = Mathf.Atan2(dir.y, dir.x) * Mathf.Rad2Deg;
    if (prevAngle != 0f) {
      selectedObject.transform.Rotate(
        Vector3.up, -(angle - prevAngle), Space.World);
    }
    prevAngle = angle;
  }

  void HandleSingleTap() {
    if (Input.GetTouch(0).phase != TouchPhase.Began) return;
    Ray ray = Camera.main.ScreenPointToRay(Input.GetTouch(0).position);
    if (Physics.Raycast(ray, out RaycastHit hit)) {
      selectedObject = hit.collider.gameObject;
      SelectionHighlight.Show(selectedObject);
    }
  }

  public void DeleteSelected() {
    if (selectedObject != null) {
      UndoStack.Instance.Push(selectedObject);
      Destroy(selectedObject);
      selectedObject = null;
    }
  }
}
using UnityEngine;
using UnityEngine.UI;
using System.Collections.Generic;

public class ObjectSelector : MonoBehaviour {

  public static ObjectSelector Instance;

  [System.Serializable]
  public struct PlaceableObject {
    public string name;
    public GameObject prefab;
    public Sprite thumbnail;
  }

  [SerializeField] List<PlaceableObject> objects;
  [SerializeField] Transform buttonContainer;
  [SerializeField] GameObject buttonPrefab;

  private int selectedIndex = 0;

  void Awake() {
    Instance = this;
    BuildUI();
  }

  void BuildUI() {
    for (int i = 0; i < objects.Count; i++) {
      int idx = i; // capture for lambda
      var btn = Instantiate(buttonPrefab, buttonContainer);
      btn.GetComponentInChildren<Image>().sprite = objects[i].thumbnail;
      btn.GetComponent<Button>().onClick.AddListener(() => Select(idx));
    }
  }

  public void Select(int index) {
    selectedIndex = index;
  }

  public GameObject GetSelected() {
    return objects[selectedIndex].prefab;
  }
}
using UnityEngine;
using System.Collections.Generic;

// Command pattern undo stack
public class UndoStack : MonoBehaviour {

  public static UndoStack Instance;
  private Stack<GameObject> history = new Stack<GameObject>();

  void Awake() => Instance = this;

  // Called after each object is placed
  public void Push(GameObject obj) {
    history.Push(obj);
  }

  // Undo button handler
  public void Undo() {
    if (history.Count == 0) return;
    var obj = history.Pop();
    if (obj != null) Destroy(obj);
  }

  public void ClearAll() {
    while (history.Count > 0) {
      var obj = history.Pop();
      if (obj != null) Destroy(obj);
    }
  }
}
System Design

Architecture Overview

AR Foundation Layer
  • ARSession — manages lifecycle
  • ARCameraManager — camera feed
  • ARPlaneManager — surface detection
  • ARRaycastManager — hit testing
  • ARAnchorManager — world locking
Application Layer
  • PlacementController — place logic
  • GestureManager — touch input
  • ObjectSelector — UI / prefab mgmt
  • UndoStack — command history
  • SelectionHighlight — visual feedback
Unity Scene Hierarchy
  • AR Session Origin (root)
  • AR Camera (child)
  • Placement Indicator (prefab)
  • Placed Objects (dynamic children)
  • Canvas UI (overlay)
Android Build Config
  • Min SDK — Android 7.0 (API 24)
  • ARCore supported device check
  • Camera permission at runtime
  • IL2CPP scripting backend
  • ARM64 target architecture
Problem Solving

Challenges & Solutions

⚠ Challenge

Objects placed on detected planes would drift slightly after placement as ARCore refined its understanding of the environment geometry.

✓ Solution

Attached an ARAnchor at the exact hit pose and parented the placed object to the anchor. ARCore continuously updates anchor poses to stay world-locked.

⚠ Challenge

Multi-touch gestures (pinch + rotate) conflicted with the single-tap placement logic, causing unintended object placements during scaling.

✓ Solution

Added an Input.touchCount guard in PlacementController — placement only fires when exactly 1 touch is active, preventing any multi-touch interference.

⚠ Challenge

Frame rate dropped below 30fps on mid-range devices when placing more than 5 high-poly objects due to excessive draw calls.

✓ Solution

Applied LOD groups on all prefabs, enabled GPU instancing on materials, and used StaticBatchingUtility.Combine() for non-interactive objects. Maintained 60fps up to 10 objects.

Get Started

Setup & Run

1

Clone the Repository

Run git clone https://github.com/tomarshaab01/ar-object-placement.git and open the project folder in Unity Hub.

2

Unity Version & Packages

Open with Unity 2022.3 LTS. In Package Manager install: AR Foundation 5.x, ARCore XR Plugin, and XR Interaction Toolkit.

3

Android Build Settings

Switch platform to Android. Set Minimum API to 24, Scripting Backend to IL2CPP, Target Architecture to ARM64. Enable ARCore Supported in XR Plug-in Management.

4

Build & Deploy

Connect an ARCore-compatible Android device. Click Build and Run. Grant camera permission on first launch and point at a flat surface to start detecting planes.
